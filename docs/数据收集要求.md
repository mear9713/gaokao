# 数据收集要求

> 高考志愿 AI Agent · 数据采集需求清单 · 2026-05-28

---

## 一、总览

| 数据类型 | 量级 | 更新频率 | 主要来源 |
|---------|------|---------|---------|
| 院校基础信息 | ~3000 所 | 每年 1 次 | 教育部 / 各校官网 |
| 专业信息 | ~500 专业 × N 校 | 每年 1 次 | 各校教务处 |
| 录取数据 | 3000 校 × 31 省 × N 年 | 每年（高考后）| 各省考试院 / 第三方 |
| 招生计划 | 3000 校 × 31 省 | 每年（5 月）| 各校招生网 |
| 保研政策 | 重点 200 校 | 每年（9 月）| 各校教务处文件 |
| 培养方案 | 重点 500 校 | 2-3 年 | 各校教务处 |
| 奖学金 / 转专业 | 重点 200 校 | 每年 | 各校官网 |

### 6. 学生输入数据（前端产生，不需采集）

供参考，画像包含的字段：

```
省份、分数、位次、选科（多选）、目标城市（多选）、
```

### 7. 知识库文档（RAG 用，非结构化）

8 类文档统一存到向量库：

| 类型 | 内容 |
|------|------|
| 招生数据 | 各校在各省招生计划文件 |
| 录取数据 | 历年录取分数线/位次汇编 |
| 保研政策 | 推免实施细则 PDF |
| 专业信息 | 专业培养目标文档 |
| 培养方案 | 课程体系 PDF |
| 奖学金政策 | 奖助学金管理办法 |
| 转专业政策 | 转专业实施办法 |
| 院校官网 | 院校概况、新闻稿 |

**每个文档需要的字段**：
- 文档 ID、类型、标题、出处（机构 / 文件号）、年份、关联院校（可空）、正文、切片+向量

**切片建议**：每块 300-500 字，重叠 50 字
**Embedding 模型**：`BAAI/bge-large-zh-v1.5`（中文 1024 维）

## 三、数据来源

| 来源 | 适用数据 | 获取方式 |
|------|---------|---------|
| 教育部高校名录 | 院校基础信息 | 官网下载 |
| 教育部学科评估 | 优势学科 | 官网下载 |
| 软科 / USnews | 院校排名 | 官网公开 |
| 各省考试院 | 录取数据 | 每年 8 月发布 |
| 各校招生网 | 招生计划 | 爬虫 |
| 各校教务处 | 保研政策、培养方案 | PDF 下载 + 解析 |
| 各校学生处 | 奖学金政策 | 官网 |
| 第三方付费 | 录取数据完整版 | 圆梦志愿 / 高考志愿君 |

**采集工具建议**：
- 网页爬虫：Scrapy / Playwright
- PDF 解析：pdfplumber / PyMuPDF
- 关键 200 校重点数据：建议人工录入或半人工校对

## 四、数据质量要求

| 维度 | 标准 |
|------|------|
| **完整性** | 985 + 211 院校核心字段必须 100% 完整 |
| **准确性** | 录取分数偏差 < 0.5%；保研率偏差 < 1% |
| **时效性** | 招生数据每年 6 月前完成更新 |
| **一致性** | 院校名称统一（"清华" / "清华大学" 不能混用）|

---

## 五、最小可用数据集（MVP）

第一阶段联调只需要：

- **院校**：100 所 985 + 211（核心字段完整即可）
- **专业**：每校 3-5 个热门专业（计算机、电子信息、自动化）
- **录取数据**：上述 100 校 × 31 省 × 近 3 年
- **保研政策**：上述 100 校的整体保研率 + 学院保研率
- **知识库文档**：每校 2-3 篇关键文档（招生简章 + 推免细则）

约 **5000 条结构化数据 + 500 篇 RAG 文档**，足够支撑 MVP 演示。
